# Home Assistant Integration

Primary integration is ESPHome native API. MQTT is supplementary and optional.

Updated design intent:

- Home Assistant is supervisory, not the real-time decision-maker
- the device evaluates the local rule table and performs the final actuation decision
- Home Assistant may write thresholds, profiles, modes, and enable states into the device
- the device continues using the last valid stored values if Home Assistant disconnects

## Entity Naming

Use a stable device prefix such as `greenhouse_controller_*`.

Examples:

- `greenhouse_controller_high_air_temperature`
- `greenhouse_controller_window_position`
- `greenhouse_controller_manual_mode`
- `greenhouse_controller_network_mode`

## Integration Notes

- Home Assistant availability must not disable local automation
- unavailable sensors should show unavailable, not stale-valid values
- diagnostics should make recovery states explicit

## Helper / YAML Pattern Guidance

Preferred pattern:

- persist device-owned values on the device through ESPHome entities
- use Home Assistant helpers for profiles and friendly operator presets
- use Home Assistant automations to copy helper values into the device
- keep Home Assistant dashboards and notifications supervisory rather than actuation-critical

See [docs/home-assistant-implementation.md](/home/ww/src/Greenhouse-Automation-Waveshare-ESP32-S3/docs/home-assistant-implementation.md) for the full entity map, helper guidance, example automations, and dashboard recommendations.

## Current Implementation Notes

- native API is the primary Home Assistant path
- MQTT is configured as supplementary telemetry and should not be treated as the source of truth for control
- the local ESPHome web server is currently the first-pass local maintenance surface
- display entities are configuration placeholders until the exact display hardware is chosen
- entity names currently follow the device-friendly-name pattern generated by ESPHome; object IDs should remain stable through the device slug
- full rule-table editing, 7-day local event log exposure, and configuration-origin metadata remain planned implementation work
